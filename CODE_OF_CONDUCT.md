# Code of Conduct

## Our Pledge

We are committed to providing a friendly, safe, and welcoming environment for all users and participants, regardless of age, body size, disability, ethnicity, sex characteristics, gender identity and expression, level of experience, education, socio-economic status, nationality, personal appearance, race, religion, or sexual identity and orientation.

## Our Standards

### Expected Behavior

- Using welcoming and inclusive language
- Being respectful of differing viewpoints and experiences
- Gracefully accepting constructive criticism
- Focusing on what is best for the community
- Showing empathy towards other community members
- Providing accurate and helpful technical guidance

### Unacceptable Behavior

- The use of sexualized language or imagery and unwelcome sexual attention or advances
- Trolling, insulting/derogatory comments, and personal or political attacks
- Public or private harassment
- Publishing others' private information without explicit permission
- Using the model or tools to generate malicious content, exploit code, or attack vectors
- Deliberately spreading misinformation about security practices
- Other conduct which could reasonably be considered inappropriate in a professional setting

## Scope

This Code of Conduct applies to all project spaces, including:

- GitHub repository (issues, pull requests, discussions)
- Hugging Face model pages and discussions
- Any deployed instances of the model
- Communications related to the project

## Responsible AI Use

### Security Considerations

When using cloud-sec-architect-8b:

1. **Verify all recommendations** against official vendor documentation
2. **Do not use** for generating exploit code or attack methodologies
3. **Do not rely** solely on AI outputs for critical security decisions
4. **Always involve** qualified security professionals in architecture reviews
5. **Report** any potential misuse or security concerns

### Ethical Guidelines

- Use the model for legitimate security architecture and compliance purposes
- Do not attempt to extract training data or circumvent safety measures
- Respect intellectual property rights in generated content
- Be transparent about AI-generated content in professional contexts

## Enforcement

Instances of abusive, harassing, or otherwise unacceptable behavior may be reported by opening a confidential issue on the GitHub repository or contacting the project maintainer directly.

All complaints will be reviewed and investigated promptly and fairly. The project team is obligated to maintain confidentiality with regard to the reporter of an incident.

### Enforcement Guidelines

Project maintainers will follow these guidelines in determining consequences:

1. **Warning**: A private, written warning with clarity around the nature of the violation
2. **Temporary Restriction**: Temporary restriction from interaction with project spaces
3. **Permanent Ban**: Permanent removal from project spaces for repeated violations
